Hello Hexo

发表于 2017-12-08 | 分类于 玩转Hexo

​ hexo是一个快速、简洁且高效的博客框架，支持Marddown，可以将支持的类型的文件转换成静态网页。网上有很多高质量的hexo教程，但是考虑到专家盲点，大家在自己搭建博客的时候总是会遇到一些无法预料的问题，于是我决定与大家分享搭建博客的过程。

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。

专家盲点是指一些专精某个领域的专家，因为太精通，以至于他忘记了当他不是专家时所遇到的困难。

Node.js

Node.js是一个基于Chrome V8 引擎的 JavaScript 运行环境。同样的，您只需要知道Node.js是hexo的运行环境即可。

注册Github账号

打开Github创建账号即可，建议与本地配置Git的时候使用的名字和邮箱相同。不要忘记去您的邮箱验证邮箱地址。

  1. 登陆Github后依次点击右上角”+”，New repository，然后设置仓库名和仓库属性，点击Create reopsitory即可。

  2. 注意：仓库名应该设置为Github昵称.github.io，结尾的github.io不能变。

配置博客文件

在您上一步创建好的文件夹中，应该有一个名为：_config.yml的文件，用记事本打开后进行编辑。

1. Site设置，从配置文件的第四行开始。

  • title为博客名，与网站名相同。

  • subtitile为关键字，您可以添加一些对博客主要内容的描述，以便搜索引擎更好的找到您的博客。

  • descriptio为描述，我在这里添加了一句座右铭。
  • author为作者，在这里填入您自己的名字即可。

2. URL设置（既指定的博客网址），就在Site设置的下边，应该能找到如下字段。 在这一部分的第三行输入您想要的博客网址即可，格式为https://xxxxx.github.io/

3. 部署方式配置，按下ctrl+f打开查找窗口，输入Deployment找到后添加如下字段。

  • type指定部署方式，当然是我们提前配置好的Git。
  • repo指定部署地址，输入我们提前申请好的Github仓库地址。
  • branch为项目分支，输入master表示主分支即可。

注意配置文件中每一项前都有一个空格，如”title: JUST”中就有一个空格，下同。
