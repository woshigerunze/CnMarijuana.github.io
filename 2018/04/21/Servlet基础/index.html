Servlet基础

发表于 2018-04-21 | 分类于 Java Web |

Servlet 简介

Java Servlet 是运行在 Web 服务器或应用服务器上的程序，它是作为来自 Web 浏览器或其他 HTTP 客户端的请求和 HTTP 服务器上的数据库或应用程序之间的中间层。

Servlet 其实就是一个在服务器端运行的小 Java 程序，可以接受 HTTP 请求并做出相应的响应，是使用 Java 进行Web开发的基石之一。借助 Servlet 可以更方便的实现动态的 Web 资源。严格来说，Servlet 只是 javax.servlet 包中的接口，是规范；一般将实现了 Servlet 接口的 Java 程序也称之为 Servlet 。

Servlet 的特点

  • Servlet 是利用Java提供的API实现的，因此它具有Java语言的优点，跨平台、灵活易拓展等；
  • Servlet 通过多线程技术实现高效处理；
  • Servlet 采用了 Java 的安全框架，再加上容器提供的安全保障，Servlet 的安全性可靠。

Servlet 运行过程

Servlet 处理 HTTP 请求的流程如下（也叫Servlet的生命周期）：

  1. 在 Web 容器启动或者第一次收到客户端请求时创建 Servlet；
  2. 服务器调用 Servlet 的 init() 方法初始化；
  3. 服务器调用 Servlet 的 service() 方法完成对客户端请求的响应和处理；
  4. 当 Web 容器关闭时，服务器调用 Servlet 的destory() 方法对占用资源进行释放；

仔细观察可以发现 Servlet 的配置方式：

  • \<servlet> 和 \<servlet-mapping> 一一对应，都有一个用于指定 Servlet 名称的 \<servlet-name>，除此之外：\<servlet> 标签有用于指定包结构的 \<servlet-class> ，\<servlet-mapping> 有用于指定虚拟路径的 \<url-mapping> ；
  • \<servlet-name> 的值是 Servlet 的名称，通常写实现类的类名，相互对应的两个父标签中 \<servlet-name> 子标签的值应该是相同的，另外这里如果不写类名也不会出现错误（两个 \<servlet-name> 的值依然相同），Web 服务器不是通过类名去寻找或加载 Servlet；
  • \<servlet-class> 的值是 Servlet 的包结构，使用该标签说明后服务器才可以找到正确的 .java 文件并通过反射的方式创建、加载正确的 Servlet；
  • \<url-pattern> 的值是虚拟路径，例如在本例中，项目开始运行之后在浏览器地址栏输入localhost:8080/test ，ServletTEST 实例就会被创建，并且网页中内容是 “First Servlet” 。

#Servlet API

Servlet API 中接口和类虽然繁多，但是都有规律可循，再加上篇幅所限，本文中不会详细介绍每一个方法。如果您有疑惑，可以查阅 Java EE 的文档。

##Servlet 接口

前面已经说过，Servlet 是 javax.servlet 包中的接口，对实现类进行了规范。 Servlet 接口中有五个方法，分别是对应初始化的 init() 、处理请求的 service() 和消亡时的 destory() ，以及用于获取 Servlet 相关信息的 getServletinfo() 和 getServletConfig()。

Servlet 接口规定了实现类应该遵守什么样的规范以及实现类的生命周期，但是在实际开发过程中我们自定义的 Servlet 通常是继承 HttpServlet 或者 GenericServlet 两个类。

##ServletConfig

ServletConfig 是位于 javax.servlet 包中的接口，代表当前 Servlet 对象的配置信息（既 web.xml 中的内容），每一个 Servlet 都有且仅有一个 ServletConfig对象。

ServletConfig 提供了四个方法用于获取配置信息。

##GenericServlet 和 HttpServlet

###GenericServlet

一个实现了 Servlet 和 ServletConfig 接口的抽象类，实现了出了service() 之外的所有方法。自定义 Servlet 对象时，可以通过继承 GenericServlet 来简化代码。

###HttpServlet

HttpServlet 仍然是一个抽象类，并且已经实现了 service() 方法，并且针对 HTTP 1.1 中的七种请求类型添加了相应的方法：doGet()、doPost()、doPut() 等。

###HttpServletRequset 和 HttpServletResponse

这两个类分别实现了 ServletRequset 和 ServletResponse 两个接口，是对 HTTP 请求报文和响应报文的封装，方便获取报文中的信息或编辑报文。
