正则表达式

发表于 2018-01-24 | 分类于 Java |

定义

正则表达式是对字符串操作的一种逻辑公式，就是用事先定义好的一些特定字符、及这些特定字符的组合，组成一个“规则字符串”，这个“规则字符串”用来表达对字符串的一种过滤逻辑。

特点

  • 灵活性、逻辑性和功能性强
  • 阅读性较差
  • 通过正则表达式可以简化对字符串的操作

Java中与正则表达式相关的类

包名java.util.regex中有两个类，分别是表示正则表达式编译形式的Pattern类和匹配器Matcher类。

Pattern类

Pattern类用于创建一个正则表达式，在Java文档中详细记录了Java支持的正则表达式语法规则，封装了用于操作正则表达式的字段和方法。

主要方法

  1. Pattern类不对外暴露构造函数，使用Pattern.complie(String regex)方法创建对象。

  2. String patten()：返回此对象在其中编译过此模式的正则表达式。其实就是compile方法中的参数regex字符串。

  1. Matcher matcher(CharSequence input)：创建匹配给定输入与此模式的匹配器，在下一部分中一起演示。

Matcher类

​ Matcher类通过正则对象的mathcer方法与字符串关联，用于创建一个对指定字符串操作的匹配器对象，然后通过Matcher类的方法对匹配的字符串进行操作。

主要方法

  1. Mathcer类不对外暴露构造函数，通过Pattern类的matcher方法获取匹配器对象。

  2. Pattern pattern()：返回与该匹配器对应的正则类。

  3. 三个匹配函数：

    • boolean matches()：检验关联的正则对象和字符串是否完全匹配，匹配返回true，不匹配返回fasle。

    • boolean lookingat()：每次都从第一个字符开始检验是否部分匹配。

    • boolean find()：每次都从当前位置开始检验是否匹配，匹配成功将移动下次匹配的位置。

      关于这三个函数的特点和区别另起篇幅介绍。

主要应用

多亏了String类强大的功能，在多数情况下我们使用正则操作字符串时不用依次构造正则类、匹配器类，而是使用String类封装好的函数，比如匹配和替换。

使用正则表达式

对比不使用正则表达式的写法，首先代码行数上大大减小，其次更改正则表达式相比更改代码更加方便灵活。

切割

将给定的字符串按要求切割为多个字符串，通过String类的split方法实现。

以输入一句英语，将每个单词截取出来储存进一个字符串数组为例。

  • 理想状况下每两个单词间都有一个空格，此时可以用空格为正则表达式切割字符串。

  • 当有些单词间的空格不止一个时，就需要在指定正则表达式的时候使用数量词修饰。
