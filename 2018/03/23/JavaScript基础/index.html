JavaScript基础

发表于 2018-03-23 | 分类于 前端基础

JavaScript一种直译式脚本语言，是一种动态类型、弱类型、基于原型的语言，内置支持类型。它的解释器被称为JavaScript引擎，为浏览器的一部分，广泛用于客户端的脚本语言，最早是在HTML网页上使用，用来给HTML网页增加动态功能。

简单来说，JavaScript（此后也简称js）是一种运行在浏览器（不能在服务器）中的编程语言，用于实现动态的网页以增强用户的交互体验。

特点

  • 解释性：作为脚本语言，js不需要编译，只需要有解释器（支持js的浏览器）就可以运行js代码编写的程序；
  • 基于对象：js程序实现的功能大多来自脚本环境中对象的方法与脚本的互相作用；
  • 事件驱动：js以事件驱动的方式直接对客户端的输入做出响应；
  • 跨平台：几乎所有的主流浏览器都支持js的解析引擎；
  • 安全性：js语法不允许程序访问本地硬盘，因此不会造成本地数据的丢失或被篡改。

JS与其他语言的关系

Java

JS由Netscape公司的LiveScript发展而来，与Java并没有直接关系。二者在语法上略有相似。

Jscript

Jscript并不是JavaScript的简称，是微软公司推出的脚本语言。相比与js，Jscript可以通过指定方式访问特定的本地硬盘文件，但是在浏览器支持方面，js几乎碾压Jscript。

JS应用

js脚本是通过嵌入或调入在网页的HTML代码中来实现功能的，通过与HTML和CSS分工合作使网页内容充实、界面美观、交互性更强。

<script>的位置

  • 浏览器解析HTML文本时是从上到下的顺序，如果将js代码写在网页中元素之前有可能影响低配终端向用户展现网页内容的速度（浏览器先解析了js代码段，而不是需要展现给用户的网页）；
  • 使用js程序操作网页中元素时需要给元素指定ID以方便js程序找到元素，而如果在指定ID之前就先解析了js代码段，会因为找不到对应ID而报错；

综上所述，能提高用户体验并且最保险稳妥的办法是防止在</body>标签之后、</html>之前，当然这只是我个人的一点小建议。在接下来的学习总结中，我可能会为了简化演示代码而将<script>放置在</body>之前，希望您不要介意。
