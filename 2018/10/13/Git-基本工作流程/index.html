基本工作流程

上一篇中介绍了 Git 的基本工作流程中的三个步骤的每一个步骤名称和效果，本篇为您详细介绍每个步骤的效果和操作。

创建仓库

Git 使用仓库这一概念来管理和控制文件，所有在 Git 仓库中的文件都可以通过 Git 来管理。

创建仓库的方式有两种，第一种是将现有的项目导入 Git，第二种是从服务器上克隆仓库。本篇总结主要介绍第一种方式：

打开 Git Bush，进入项目目录（就是三个区域中的工作目录）后执行 git init

  • Git Bush 和 CMD 略有不同，在不同盘符切换时仍需使用 cd 命令；
  • 执行结束后会生成一个名为 .git 目录（该文件夹默认隐藏），其功能详细介绍在本系列学习总结上一篇中已详细介绍。

跟踪文件

此时 Git 仓库和暂存区都是空的，因为 Git 并没有追踪任何文件。需要注意的是 .git 文件夹此时并不为空，它保存了一些 Git 需要的资源，而 index 文件根本不存在。

追踪是文件由 Untracked 状态变为 Staged 状态的过程，Git 会扫描文件获取快照后将其存入暂存区。

使用 add 命令：

  • 参数 file 可以为一个文件或目录，如果是一个目录时会将该目录下所有文件跟踪；

  • file 为一个文件时不可省略后缀名；

  • 可使用占位符 来简写或一次跟踪多个文件，比如要跟踪项目中所有的 Java 源代码文件只需要执行 `git add .java`；

  • git add . 会将工作目录中所有文件都追踪。

在工作目录下新建一个 test.txt 文件来进行实验，追踪文件后可以使用 git status 命令来查看各文件状态。

Changes to be committed 就是指这些文件快照在暂存区了。

提交快照

提交快照的过程是将文件快照从暂存区放入 git 目录的过程，经提交的文件状态会从以暂存变成未修改。

执行git commit 命令来提交快照，执行后会打开系统默认的文本编辑器用于编辑一些提交的信息，而我设置了 Git 的默认文本编辑器为 Vim。
