Git 简介

发表于 2018-10-13 |

简介

Git 是一个分布式的版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。

​ ——百度百科

在开发之初，Git 的服务目标是帮助管理 Linux 内核开发，现在已经成为全球的开源社区和开放源码的软件的重要基石。

快照

与其他版本管理工具不同，Git 记录的是文件快照而非文件变化。

存储网络行业协会 SNIA（StorageNetworking Industry Association）对快照（Snapshot）的定义是：关于指定数据集合的一个完全可用拷贝，该拷贝包括相应数据在某个时间点（拷贝开始的时间点）的映像。快照可以是其所表示的数据的一个副本，也可以是数据的一个复制品。

可以这样理解：对一张桌子拍一张照片，记录了桌子上所有物品的摆设位置、状态，之后无论桌上的物品被打乱到什么程度，都可以对照照片将其恢复原样。这张照片就是桌上物品的快照。

SVN 等其他版本管理工具记录的是文件的变化，比如我们在一个已被管理的 txt 文件中添加了几个字符，SVN 就会记录下添加字符的内容。

Git 记录快照

使用 Git 管理 ABC 三个文件时：

  1. Version 1 时三个文件被创建，Git 会记录三个文件的快照；
  2. 后续每个版本变动时仅记录变化文件的快照：
    • Version 2：记录快照 A1、C1；
    • Version 3：记录快照 C2；
    • Version 4：记录快照 A2、B1；
    • Version 5：记录快照 B2、C3

SVN 记录差异

使用 SVN 管理 ABC 三个文件时：

  1. Version 1 时三个文件被创建，SVN 会记录下三个文件的所有内容；
  2. 后续每个版本变动时仅记录变化文件的差异：
    • Version 2：记录 A 的差异 Δ1 和 C 的差异 Δ1；
    • ……
    • Version 5：记录 B 的差异 Δ2 和 C 的差异 Δ3；

Git 工作流程

要了解 Git 的工作流程需要先理解 Git 中三个目录和四种状态的概念。

三个目录

Git 项目中有三个主要区域：

  • .git directory：Git 目录，保存了 Git 生成的数据和需要的资源；
  • Working Directory：工作目录，是一个项目的全部内容；
  • Staging Area：暂存区，一个名为 index（无后缀）的文件，保存着下次要提交的内容（提交是 Git 的一个工作流程）；
    • 暂存区也被叫做索引，但是根据该区域中文件快照的性质来说称其未暂存区更符合国人的语言习惯。

文件四种状态

加上未被 Git 管理的状态，有四种文件状态需要您了解：

  • Untracked：未被跟踪状态，Git 不会管理该文件的变化；
  • Staged：已暂存状态，使用 Git 管理一个未跟踪文件时它会先变成已暂存状态，此时文件的快照会进入暂存区；
  • Unmodified：未修改状态，在 Git 记录中没有变化的文件（Git 存储的快照和该文件对比后未发现改变）；
  • Modified：已修改状态，在 Git 记录中发生变化的文件；

Git 工作流程

请先对以下工作流程有大致了解，看完本系列总结的下一篇后您就会对 Git 管理版本的流程完全了解。

  1. 在工作区中修改文件；
  2. 修改过的文件的快照会进入暂存区；
  3. 提交更新后暂存区的内容会永久储存到 Git 仓库目录。
