HttpServletResponse

HttpServletResponse （此后或简称 Response）对象表示服务器对客户端的响应，该对象是对响应报文的封装，包括响应行、响应头和响应体三部分。与 HttpServletRequest 对象相同，我们根据不同的功能来介绍这个该类的不同方法。

设置响应码

希望您还记得 Http 协议中常见的响应码和它们对应的含义。

  • void setStatus(int sc)：设置该响应的状态码，用于无错误的情况；
  • void sendError(int sc)：使用指定状态代码并清除缓冲区将错误响应发送到客户端。

设置响应头

常用的有两个方法用于设置响应头：

  • void setHeader(String name, String value)：根据给定名称和值设置响应头，如果已经设置了该头，新的值将重写以前的值；
  • void addHeader(String name, String value)：根据给定的名称和值添加响应头，允许响应头有多个值。

此外还有 value 参数的类型为 int 的函数和专用于日期时间的函数，这里不再详细说明。

修改 MyResServlet 的 doGet 方法，启动服务器后直接在浏览器地址栏输入 localhost:8080/response 即可，之后使用抓包工具来查看验证。

如果在这两个方法的 name 参数中传入并不存在的 Http 响应头名，依然会正常添加进响应报文中，不会报错。

设置响应体

当客户端收到响应报文，会把响应体中的内容解析后展示给用户，Response 提供了两个方法供我们向响应体中添加内容。

  • ServletOutputStream getOutputStream()：返回一个字节输出流对象；
  • PrintWriter getWriter()：返回一个字符输出流对象。

这两个方法返回的输出流对象可以用来向响应体中添加自己想要的内容，浏览器会将这些内容解析后再呈现。

重定向

此处重定向是指让客户端重新跳转至目标网页，比如说在某个网站的登陆界面中，您输入了账户和密码点击了登陆的按钮，接下来会跳转到已登录界面或显示登陆失败的界面，这个过程就是一个重定向。重定向的主要有两个重点：

  1. 响应码：响应报文中的响应码应告诉浏览器它应该进一步操作才可以将正确完整的网页展示给用户，浏览器才会对重定向的操作有所反应。

  2. URL 地址：重定向是重新跳转至目标网页，那么一个新的 URL 地址肯定是必不可少的。

可以使用 setStatus() 设置响应码 和 setHeader() 设置 location 头的值来实现这个功能，也可以直接使用一个专门用于重定向的函数来实现：

  • void sendRedirect(String location)：使用指定重定向位置 URL 将临时重定向响应发送到客户端。

禁用浏览器缓存

浏览器缓存是浏览器对近期请求过的文档的储存，可以提高网页打开速度，改善用户体验等，但是有一些网站是不希望被缓存下来的，比如一些时效性极高的内容。我们可以通过设置响应报文中的三个头信息来禁用浏览器对当前网页的缓存。

  • Pragma：老版本中用来控制缓存的头，仅有一个为“no-cache”的值；
  • Cache-Control：Http 1.1 中控制缓存的头，当值为“no-cache”的时候含义为不缓存；
  • Expires：当前资源的有效时间，如果客户端时间超过该时间，则不会使用本地缓存，因为是与协调世界时的毫秒差，如果仅是为了控制缓存，方便起见可以令它的值为 -1 。

单看解释的话，这三个响应头单独任何一个都可以用来控制缓存，但是由于浏览器多种多样，各个浏览器支持的 Http 版本也不尽相同，最稳妥的办法还是把这三个头信息都加入响应报文。

解决乱码

Request 解决乱码分为两种情况，依据分别是请求方式的不同，而 Response 解决乱码也分为两种情况，分别是向响应体中添加内容的输出流是字节流还是字符流。

字节流 ServletOutputStream

首先请明确，字节流输出的内容不一定会出现乱码的情况，如果服务器端的默认编码和客户端浏览器的编码方式恰好相同就不会出现乱码情况。

而如果出现了乱码的情况，有两个步骤来解决问题：

  1. 通过响应头设置浏览器打开文件时应采用的编码；
  2. 获取字符串的 Byte 数组时指定相同的编码方式。
