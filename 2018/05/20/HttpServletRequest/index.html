HttpServletRequest 和 HttpServletResponse 是 Servlet 技术中用于处理 Http 请求和响应的类。

HttpServletRequest

HttpServletRequest （此后或简称 Resquest）对象表示客户端的请求，当客户端通过 HTTP 协议访问服务器时，会对请求信息（请求报文）中的所有内容都进行封装，该对象包括请求的地址，请求的参数，提交的数据，上传的文件客户端的ip甚至客户端操作系统都包含在其内。下面根据不同的功能分别介绍这些，不过为了简化演示代码、确保您能看懂代码，我先把这次演示代码的框架展示一下：

获取请求行内容

  • String getRemoteAddr()：返回发送请求的客户端（或最后一个代理）的 IP 地址，实际上这个方法是定义在 ServletRequest 接口中的；
  • String getMethod()：返回发送的请求方式；

获取请求头内容

请求头是 Http 请求报文中很重要的一部分内容，其中包含这客户端的各种信息，可以通过 HttpServletRequest 提供的一些方法来获取到这些信息，以方便近一步的操作。

  • String getHeader(String name)：以 String 的形式返回指定请求头的值，如果不包含该请求头则返回 null ；

  • Enumeration getHeaders(String name)：有些请求头的值可能不唯一，该方法就是用于获取这种请求头的所有值，会返回一个包含该请求头所有值的枚举；

  • Enumeration getHeaderNames()：返回一个包含所有请求头名称的枚举。

获取请求参数

请求参数是请求报文中客户端自定义的内容，通常是来自与客户在网页上填写的表单内容之类的，请求参数也是以键值对形式存在的数据。

前两个部分的内容分别对应 Http 请求报文中的请求行和请求体，这部分内容则不同——因为 Get 请求方式提交的请求报文中请求参数并不位于请求体，而是与请求行中的资源路径在一起。

  • String getParameter(String name)：以 String 形式返回请求参数的值，如果该参数不存在则返回 null，这个方法只能使用在仅有一个值的请求参数；
  • String[] getParameterValues(String name)：以 String[] 形式返回请求参数的值，如果参数不存在则返回 null ；
  • Map getParameterMap()：返回所有请求参数和对应的值。

解决乱码

不知道在上一个例子中您有没有尝试在 “输入用户名” 处输入中文，如果有的话，极有可能会得到一串乱码。在客户端发送请求，服务器发送响应的过程中，其实都有可能出现乱码问题，今天先来学习发送请求时如果出现乱码该如何解决。

Get 请求的情况

Get 请求报文的请求体是一个空行，所以用设置缓冲区编码的方式是无法解决乱码问题的，有另外两种方式可以解决。

逆向编解码

使用 java.net 包中的解码、编码对象来解决这个问题。

直接修改服务器编码

直接修改 Web 容器的编码方式当然可以解决乱码问题，在 Tomcat 安装目录中的 conf 文件中的 server.xml 是服务器的配置信息，在 \<Connector> 标签中添加一个新的属性，指明编码方式即可。

但是并不推荐使用这种方式来解决问题，因为在实际的项目中，您不一定拥有 Tomcat 服务器的完全管理权，而且 一个服务器有可能拥有多个项目，如果您为了自己的项目更改了服务器的编码方式，那极有可能导致其他的项目出现乱码。
