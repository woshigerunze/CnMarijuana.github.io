HTTP协议概述

发表于 2018-05-20 | 分类于 Java Web |

HTTP协议

超文本传输协议（HTTP，HyperText Transfer Protocol）是互联网上应用最为广泛的一种网络协议。

简介

  • 超文本：表明通过 HTTP 可以传输的资源不止包括文本，这也是能在互联网上查看图片的基础；
  • 传输：所谓“上网”，不过是将本地请求上传，服务器资源下载；
  • 协议：客户端连接服务器后，如果想要获取服务器中的资源，需要遵守一定的通讯格式，HTTP 协议定义了客户端和服务器的通讯格式。

特点

  • 基于 TCP/IP 链接而建立；
  • 默认端口 80；
  • 基于请求和响应的模式。
    • 先有请求，后有响应；
    • 没有请求无法响应。

#HTTP报文

报文即为在 HTTP 协议规范下客户端访问服务器端资源时双方互相发送的内容，分为客户端发送给服务器的请求报文和服务器发送给客户端的响应报文。

##请求报文

HTTP 协议是先请求后响应，那我们当然也要先学习请求报文。为了简化演示代码，我使用了一个由 IDEA 创建的未做任何修改的 JavaWeb 项目、Tomcat 做本地服务器、Fidder 4 抓取 HTTP 请求报文的内容。

Fidder 4 是一个专业的抓包工具，因为普通情况下我们是无法看到请求和应答的内容，需要借助抓包工具来实现。常见的抓包工具还有专供 IE 浏览器的 HttpWacth 插件和火狐浏览器的 Firebug 插件，但是前者免费版功能较少，而后者已经不能在最新的火狐浏览器上运行。如果您想在这方面钻研一番，我推荐您使用 Fidder 4。

这段“代码”就是您在浏览器地址栏中输入 http://localhost:8080/index.jsp 浏览器发送给服务器的请求报文，分为请求行和请求头（POST 提交方式多了请求体内容）。

注：无论什么提交方式，HTTP 协议都规定请求头之后有一个空行。

本例中使用 GET 提交方式，没有请求体，在介绍 GET 和 POST 提交方式的区别时再详细介绍请求体。

请求行

请求行用于描述客户端的请求方式、请求的资源名称，以及使用的 HTTP 协议版本号。是上面第一行的内容，格式为：请求方式 资源路径 协议版本；

  • HTTP 协议中常用的请求方式有 GET 和 POST，此外还有 HEAD、OPTIONS、DELETE、TRACE、PUT、CONNECT；
  • 资源路径既为所请求资源的具体路径，一般情况下是 URL 链接；
  • HTTP 协议的版本有 1.0 和 1.1，具体区别下面详细说明。

###请求头

请求头是第一行请求头之后空行之前的内容，是以键值对形式存在的内容，用于描述客户端请求哪台主机，以及客户端的一些环境信息等。

由于之前打开的是一个很简单的无任何内容的网页，上面请求报文中请求头比较少，事实上请求头的内容有很多，可以分为六个种类：Cache、Client、Cookie/Login、Entity、Miscellaneous和Transport。Fidder 4会将请求头的内容按照这六种分类展示。

####Cache

  • If-Modified-Since：对应的值为本地缓存的最后修改时间（精确到分），和Last-Modified一起使用控制缓存；
  • If-None-Match：对应的值为ETag（网站中每个资源都有对应的ETag，可以理解为一个ID），当用户请求某个资源时，如果本地的ETag（本地有ETag的条件是访问过该资源并且有缓存）和服务器端不同才会从服务器端下载，相同直接打开本地缓存。

####Client

  • Accept 浏览器可以识别的资源类型；
  • Accept-Language 客户端语言；
  • Accept-Encoding 浏览器接收的编码方式，通常是压缩方式；
  • User-Agent：客户端使用的操作系统和浏览器名称版本；

####Cookie/Login

  • Cookie：将cookie的值发送给服务器，可以说是最请求头最重要的内容之一；

####Entity

  • Content-Length：发送给服务器的数据长度。

####Miscellaneous

  • Referer：提供了Request的上下文信息的服务器，告诉服务器我是从哪个链接过来的，可以通过这个实现防止盗链的功能。

####Transport

  • Connection：决定请求完成后客户端和服务器端的连接是否关闭；
    • keep-alive：不关闭；
    • close：关闭。
  • Host：请求的服务器地址。

响应报文同样分为响应行、响应头和响应体，同样的响应头和响应体之间也有一个空行。

响应行

响应行用于描述服务器对请求的处理结果。是响应报文中的第一行，内容是响应报文的 HTTP 版本和响应码，格式为协议版本 响应码。

响应码和对应的含义

响应码 对应内容
100~199 表示成功接收请求，要求客户端继续提交下一次请求才能完成整个处理过程
200~299 表示成功接收请求并已完成整个处理过程，常用200
300~399 为完成请求，客户需进一步细化请求。常用302、307和304
400~499 客户端的请求有错误，常用404
500~599 服务器端出现错误，常用 500

响应头

第一行响应行之后、空行之前的内容，同样是以键值对形式存在的重要信息。

响应头的内容同样分为六个部份。

Cache

  • Date：生成报文的时间；
  • Expires：返回本地缓存的有效时间，如果客户端时间超过该时间，则不会使用本地缓存；
  • Pragma：只有一个对应值no-cache，请求头中有该内容时网页不会缓存；

  • Cache-Control：指定网页的缓存方式；

    • Public ：可以被任何缓存所缓存；

    • Private：内容只缓存到私有缓存中；

    • no-cache：所有内容都不会被缓存，该功能和Pragma重复，事实上Pragma是老版本中控制缓存的唯一方式，为了保证向下兼容在HTTP 1.1中保留了Pragma。

####Cookie/Login

  • Set-Cookie：用于把cookie 发送到客户端浏览器， 每一个写入cookie都会生成一个Set-Cookie。

Entity

  • ETag：和If-None-Match 配合使用；
  • Last-Modified：用于指示资源的最后修改日期和时间，和If-Modified-Since一起实现控制缓存的功能；
  • Content-Type：用于告知浏览器响应对象的类型和编码方式；
  • Content-Length：指明响应的实体正文的长度；
  • Content-Encoding：指明服务器使用的压缩方法（gzip，deflate）。

Miscellaneous

  • Server：指明服务器的种类和版本。

Transport

  • Connection：与请求头中的作用相同，属性值也相同。

Location

  • Location：用于重定向一个新的位置，包含新的URL。
