ServletContext

发表于 2018-05-20 | 分类于 Java Web |

ServletContext 是 Servlet API 中最大的接口，同样位于 javax.servlet 包中，呈现了 web 应用的 Servlet 视图。

简介

当 Web 容器启动时，会为每一个 Web 应用创建一个 ServletContext 对象，该对象表示当前 Web 应用，也被称为域对象。

  • 一个 Web 应用对应一个 ServletContext 对象；
  • 一个 Web 应用下有多个 Servlet 对象，这多个 Servlet 对象共享同一个 ServletContext 对象。

功能

ServletContext 提供了很多方法，分别对应不同的功能，下面分功能介绍方法。

配置

首先需要学习如何配置 Web 应用的全局初始化参数，在配置文件 web.xml 文档中 \<web-app> 根标签下使用 \<context-param>添加参数，与 \<init-param> 相同，同样使用\<param-mane> 和 \<param-value> 分别添加键和值。

例如，编码格式和使用服务器类型等信息需要在 Web 应用启动前就获取到，所以这类信息往往使用 \<context-param> 添加进配置文档。

获取

基本与 servletConfig 相同：

  • String getInitParameter(String name)：根据指定键名返回对应的值；
  • Enumeration getInitParameterNames()：返回迭代器，可以通过该迭代器依次获取到键名，再通过前一个方法获取对应的值。

数据共享

其实是在多个 Servlet 对象间进行数据共享，例如一个统计网页访问次数的功能，需要有两个 Servlet 分别负责统计次数和展示数据，共用的是同一个数据，可以将该数据存入 ServletContext 对象中。

  • void setAttribute(String name, Object obj)：按键值对的方式将数据存入 ServletContext 对象中；
  • void removeAttribute(String name)：删除指定的键和对应的值；
  • Object getAttrubute(String name)：根据指定的键获取对应的值。
