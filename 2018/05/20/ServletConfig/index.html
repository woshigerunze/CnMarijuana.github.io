ServletConfig

发表于 2018-05-20 | 分类于 Java Web |

ServletConfig 是对 javax.servlet 包中的一个接口，定义了获取 Servlet 对象配置信息的规范，在 GenericServlet 中以进行了初步实现。

Servlet配置文件

有必要说明的是，在 web.xml 的配置文件中，可以配置的 Servlet 信息不仅局限于上例中的名称、包结构和虚拟路径， \<servlet> 中还有用于提供初始化信息的 \<init-param> 。\<init-param> 标签中有两个子标签，分别是 \<param-mane> 和 \<param-value>，两个子标签的值分别对应键和值。

假设 SecondServlet 标签需要与数据库进行交互，它肯定需要知道访问数据库的用户名和密码，将这些信息写入配置文件后，就可以在 SecondServlet 的 init() 方法中使用 ServletConfig 的相关方法获取到配置文件的信息，方便下一步实现登陆数据库的功能。

当然，为了简化演示代码，我只是简单的配置文件中的信息获取后输出在网页上。

ServletConfig

ServletConfig 是一个很简单的接口，仅有四个方法，下面挨个介绍。

  • String getServletName()：返回当前 Servlet 的名称；
  • String getInitParameter(String name)：根据指定的键返回配置文件中对应的值；
  • Enumeration getInitParameterNames()：返回一个用于遍历配置文件中 \<param-name> 的迭代器。

个人不是很明白 Servlet 的 getInitParameterNames() 方法为什么要返回远古时代的 Enumeration 迭代器。
