XML 和 DTD 约束简介

发表于 2018-05-20 | 分类于 Java Web |

XML 简介

xml 全称为 Extensible Markup Language，意味可拓展标记型语言。作为标记型语言，xml 也依靠标签来组成文档，它与 html 主要有以下几点不同：

  • xml 的标签没有预定义，使用时需要自定义标签，而 html 的标签都是预定义好的；
  • xml 被设计为传输和储存数据，旨在传输信息，关注点是数据的内容；
  • html 被设计为显示数据，旨在显示信息，关注点是数据的外观。

XML的应用

xml的主要应用如下：

  • xml 用于在不同系统和设备之间进行数据的传输；
  • xml 用来表示有关系的数据；
  • xml 经常用在程序的配置文件中。

XML的语法

xml 的文档都是储存在后缀名为 xml 的文件中的，使用记事本等普通的文本编辑器即可查看或编辑 xml 文档。普通浏览器就带有解析xml文本的功能，您可以使用常用的浏览器来检测并改进您自己的 xml 文档。

文档声明

与 html 相同，xml 也有文档声明，需要在文档声明的中告知解析器该文档是 xml 类型的文档，使用的 xml 版本和编码格式。

  • version：xml 版本，一般情况下使用 1.0；
  • encoding：该文档的编码格式；
    • gbk：汉字内码扩展规范，后缀 2312 表示简体中文，2310 繁体字符；
    • utf-8：一种针对 Unicode 的可变长度字符编码，又称万国码；
    • ISO-8859-1：对欧洲使用语言支持的编码表。
  • standalone：该文件是否独立；
    • yes：该文档独立，不能引用外部的 DTD 文件（下面介绍DTD）；
    • no：该文档非独立，可以引用外部的 DTD 文件。
  • 文档声明必须写在整个文档的第一行第一列（之前不会有换行和空格）。

注意：如果您使用记事本软件编辑xml文档并将编码方式指明为utf-8，在保存文件时需要点击“另存为”然后将编码方式更改为 utf-8，因为记事本的默认编码方式是ANSI。

定义标签

标签基本语法

  • 与 html 相同，一个 xml 文档必须有且仅有一个根标签，其他所有标签都是该标签的子标签，根标签名并无要求，如<label></label>;
  • xml 标签必须正确结束：
    • 标签内有内容时：<label>...</label>;
    • 标签内无内容时可以采取自闭标签的格式：<label/>。
  • xml 标签允许嵌套并且必须正确嵌套：
    • 正确嵌套：<Flabel><Clabel></Clabel></Flabel>;
    • 错误嵌套：<Flabel><Clabel></Flabel>。

标签定义语法

  • xml 标签对英文字母的大小写敏感，<a>和<A>不是同一个标签；
  • xml 标签运行使用中文字符做标签名（但是并不推荐）；
  • 标签名不能以数字和下划线开头；
  • 标签名不能以 xml、XML 字母组合等开头；
  • 标签名不能包含空格和冒号。

定义属性

  • 一个标签上可以有多个属性但是属性名不能相同；
  • 属性名和属性值之间用 “=” 相连，属性值包含在引号（单双皆可）内；
  • 属性名的命名规范和标签的命名规范相同。

注释

  • 与 html 相同，xml 的注释语法为<!--注释-->；
  • xml 解析器并不会忽略注释代码，所以与 html 相比有一些不同之处：
    • xml 注释不允许嵌套；
    • xml 注释不能出现在文档第一行，第一行第一列必须是文档声明。

特殊字符

如果需要将一个特殊的字符储存在 xml 的数据中（通常是 xml 语法中有特殊规则的字符），不能将其直接写入，否则会被解析器错误解析。而想要添加这些特殊字符时，需要使用特殊的转移字符。

特殊字符 < > & ' "
转移字符 &lt; &gt; &amp; &apos; &quot;

虽然 XML 的标签可以在命名规则允许下自定义，但有时候我们仍然需要对标签的名称规定书写规范，将其称之为XML 的约束。目前常用的约束方式有 DTD（document type definition 文档类型定义）和 Schema，下面主要介绍 DTD 约束方式。

DTD约束使用方式

创建并编写DTD文件

  1. 创建一个后缀名为 dtd 的文本文件，使用普通的文本编辑器即可打开编辑；
  2. xml 文档中有多少种元素（标签），就在dtd文档中写多少个约束列表（<!ELEMENT 元素名 xxxx>）；
  3. 根据元素的类型在dtd文档中分别编辑对应的约束列表：
    • 简单元素（无子标签的元素）：<!ELEMENT 元素名 约束规则>；
    • 复杂元素（有子标签的元素）：<!ELEMENT 元素名 (子元素列表)>。

关联DTD和xml

  • 引入外部 dtd 文件：<!DOCTYPE 根标签名 SYSTEM "dtd文件路径">；
  • 在xml文件内部编写 dtd 文档 ：<!DOCTYPE 根标签名 [元素列表]>；
  • 使用网络上的 dtd 文件：<!DOCTYPE 根标签名 PUBLIC "名称" "URL链接">。

元素约束

其实上面已经提到了部份 DTD 约束 XML 标签的方式，下面对于约束规则做详细总结：

  • 简单元素的约束规则：
    • (#PCDATA)：表示该元素中内容为字符串；
    • EMPTY：表示该元素中不能有内容；
    • ANY：表示该元素中可有任意内容。
  • 复杂元素的子标签修饰符：
    • +：表示该子标签至少出现一次；
    • ?：表示该子标签出现零次或者一次；
    • *：表示该子标签可出现任意次；
    • 无修饰表示只能出现一次。
  • 复杂元素的子标签列表中的间隔符号：
    • ,：两个标签的顺序与列表中顺序相同；
    • |：两个标签只能出现其中一个

属性约束

dtd 可以对标签和内容进行约束，当然也可以对标签的属性进行约束。其语法格式为<!ATTLIST 标签名 属性名 属性类型 属性约束>。

  • 属性类型：
    • CDATA：字符串类型；
    • ID：由字母和下划线开头组成的字符串，但是一个XML文档中只能出现一次；
    • 枚举：表示该属性值只能从指定的范围中选择且只能选择一个。
  • 属性约束：
    • #REQUIRED：表示该属性必须存在且存在一次；
    • #IMPLIED：表示该属性可有可无；
    • #FIXED：表示该属性为固定值；
    • 默认值：表示该属性如果不指明属性值就会使用默认值做属性值。

定义和使用实体

可以在 dtd 文档中定义实体，并在 xml 文档中使用。

  • 定义实体：<!ENTITY 实体名 实体值>；
  • 使用实体：&实体名;。

比如可以将某个特定的日期在dtd文档中定义，然后在xml文档中直接使用实体名，当xml文档被解析时实体名会被替换成实体值。

定义了实体的 dtd 文档最好编写在 xml 文档的内部（第二种关联方式），不然在浏览器中解析查看时会出现问题（ ie 直接将标签 \<a> 显示为自闭标签 \<a/> ，Firefox 提示“未定义的实体”）。
