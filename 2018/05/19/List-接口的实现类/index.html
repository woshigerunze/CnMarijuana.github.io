List 接口的实现类

发表于 2018-05-19 | 分类于 Java |

List 接口

有序的 collection（也称为序列）。此接口的用户可以对列表中每个元素的插入位置进行精确地控制。用户可以根据元素的整数索引（在列表中的位置）访问元素，并搜索列表中的元素。

通俗的说 List 接口的实现类有以下几个特点：

  1. 存入和取出的顺序一致；
  2. 元素都含有角标索引；
  3. 元素可以重复。

List 接口的实现类

ArrayList

该类是实现了 List 接口的大小可变的数组，实现了所有可选列表操作，并且可以储存包括 null 在内的所有元素。

构造方法

  • ArrayList()：构造一个默认长度的空列表（默认长度为10）；
  • ArrayList(Collection<? extends E> c)：构造一个包含指定 collection 的元素的列表，排列顺序与原来的 collection 相同。
  • ArryList(int initialCapacity)：构造一个指定长度的空列表。

常见方法

按照对元素的添加、删除、修改和获取多种不同的功能将常用的方法分类介绍。

添加

  • boolean add(Object obj)：将一个对象添加到容器中，添加成功返回 ture，失败返回 false；
  • boolean addAll(collection coll)：将另一个容器中所有对象都添加的容器中，同样的失败时返回 false；

删除

  • void clear()：删除容器中所有对象；
  • boolean remove(Object obj)：删除指定对象；
  • boolean removeAll(collection coll)：删除两个容器中相同对象，保留不同的对象；
  • boolean retainAll(collection coll)：保留两个容器中相同对象，删除不同对象；

判断

  • boolean ifEmpty()：集合为空返回true；
    boolean contains(Object o)：如果此 collection 包含指定的元素，则返回true。

获取

  • int size()：返回容器长度

LinkedList

该类是实现了 List 接口的链表。除了实现 List 接口外，LinkedList 类还为在列表的开头及结尾 get、remove 和 insert 元素提供了统一的命名方法。这些操作允许将链接列表用作堆栈、队列或双端队列。

构造方法

  • LinkedList()：构造一个空列表；
  • LinkedList(Collection<? extends E> c)：构造一个包含指定 collection 中的元素的列表，这些元素按其 collection 的迭代器返回的顺序排列。

常见方法

同样按照对元素的添加、删除、修改和获取多种不同的功能将常用的方法分类介绍。

添加

  • void add(int index, E element)：在指定位置（index）添加指定元素（element）

  • boolean addAll(int index, Collection<? extends E> e)：在指定位置插入另一个 collection 集合所有元素

  • void addFirst(E e)：将指定元素插入此列表的开头

  • void addList(E e)：将指定元素插入此列表的结尾

删除

  • E remove(int index)：删除并返回指定位置元素
  • E removeFirst()：移除并返回此列表的第一个元素
  • E removeLast()：移除并返回此列表的最后一个元素

修改

  • E set(int index, E element)：将指定位置（index）的元素换为element，并返回原来的元素

获取

  • Object get(int index)：返回指定位置的元素

  • int Indexof(E element)：返回指定元素的位置，如果不包含指定元素返回-1

  • E getFirst()：返回此列表中头部位置的元素

  • E getLast()：返回此列表中尾部位置的元素

  • int lastIndexof(E element)：返回此列表中最后出现的指定元素的索引；如果列表不包含此元素，则返回 -1。

  • List SubList(int start, int end)：返回列表的子列表，包含start位置的元素不包含end位置元素

区别辨析

ArrayList 实质上是数组，LinkedList 实质上是双向链表，所有它们都具有原数据结构的特点；

  • ArrayList 查询快、添加和删除慢；
  • LinkedList 查询慢、添加和删除快；

根据这些特点可以判别使用哪个 List 更合适。

迭代器

迭代器 Iterable 是 Java 提供的用于方便数据依次获取的接口，可以使用该类进行增强 for 循环（foreach 语句）的目标，不过篇总结中暂时不会出现有关增强 for 循环的内容，具体内容会在以后的部分介绍。

获取迭代器对象

  • Iterator\<E> iterator()：返回按适当顺序在列表的元素上进行迭代的迭代器。

该方法是在 List 接口中定义的，所以 ArrayList 和 LinkedList 类都已经实现了对应方法，可以直接使用。

使用迭代器
