Set 接口和实现类

发表于 2018-05-19 | 分类于 Java |

Set接口

一个不包含重复元素的 collection。 更确切地讲，set 不包含满足 e1.equals(e2) 的元素对 e1 和 e2，并且最多包含一个 null 元素。

Set 接口主要有以下几个特点：

  1. 元素不能重复

  2. 不保证有序

  3. 继承了Collection的方法

Set 接口的实现类

Set 接口也是继承自 collection 接口，所以和 List 有很多共通的方法，这里就不再详细介绍了，本篇总结主要介绍 Set 接口和它的实现类与 List 接口不同的地方。

HashSet

此类实现 Set 接口，由哈希表（实际上是一个 HashMap 实例）支持。

哈希表

  • 一种数据结构，利用特定的算法（类的hashCode方法）对元素进行计算决定储存的位置；

  • 哈希表判断两个元素是否相同的过程：

    1. 判断两个元素的哈希值是否相同（hashCode方法的返回值），如果相同进行第二步，不同表示两个元素非相同元素；
    2. 判断两个元素的内容是否相同(equals方法)。

用HashSet储存元素

  • Java 类库中的类

  • 自定义类

在哈希表中储存自定义类时，需要复写 hashCode() 和 equals() 方法，便于hashSet判断是否为相同元素。

可以看出输入的顺序和储存的顺序并不相同。

子类LinkedHashSet

具有可预知迭代顺序的 Set 接口的哈希表和链接列表实现。此实现与 HashSet 的不同之外在于，后者维护着一个运行于所有条目的双重链接列表。此链接列表定义了迭代顺序，即按照将元素插入到 set 中的顺序（插入顺序）进行迭代。注意，插入顺序不 受在 set 中重新插入的 元素的影响。

即为：LinkedHashSet 的哈希算法决定了该容器的元素可以确定储存顺序（非与输入顺序相同），是一个使用哈希表的双重链表。

TreeSet

一个可以根据字典顺序存储输入元素的集合，或者也可以根据创建该对象时提供的 Comparator 排序。

  • 字典顺序：也叫自然顺序，比如在字典中 abandon 在 a 之后一样；
  • Comparator：强制给某个 collection 对象进行整体排序的比较函数，来自于同名接口。

构造方法

  • TreeSet()：构造一个根据默认字典顺序排序的容器

  • TreeSet(Comparator<? super E> comparator)：构造一个根据指定比较器进行排序的容器

区别辨析

HashSet 和 TreeSet 是 Set 接口的实现类，因此都具有 Set 接口的特点，同时又都有各自的特点。

  • 相同点
    • 单列集合，元素不可重复
  • 不同点
    • HashSet无序，TreeSet有序
    • 存储时保证数据唯一性依据不同：HashSet 通过 equals() 和 hashcode() 方法，而 TreeSet 通过 compareTo 方法。
